CSS Layout and Responsive Design

How I learned to style webpages and make them look good on every screen size.

What Is CSS and Why Does It Matter?

CSS stands for Cascading Style Sheets. If HTML is the skeleton of a webpage, then CSS is everything that makes it look the way it does — the colors, fonts, spacing, and layout. We started learning CSS in Week 3, and it quickly became the part of the course I spent the most time on.

One of the most important rules in this course was to always use an external stylesheet. That means all your CSS goes in a separate .css file and gets linked to your HTML pages using a <link> tag in the <head>. No inline styles, no internal style blocks — external only. At first this felt like extra work, but once I had multiple pages sharing the same stylesheet it made total sense.

Core CSS Concepts

Selectors, Properties, and Values

CSS works by targeting HTML elements with a selector and then applying style rules to them. For example, p { color: #333; font-size: 1rem; } targets all paragraph tags and sets their color and size. You can target by element type, class name, or ID. Week 3 covered the difference between these and when to use each one.

Colors and Fonts

Week 5 went deeper into color and typography. I learned how to use Google Fonts to bring in custom typefaces that look much better than the browser defaults. For colors, I used CSS custom properties (variables) to define my color palette once at the top of my stylesheet and reuse them everywhere — which made it easy to stay consistent across all pages.

The Box Model

Every HTML element is a box made up of four layers: content, padding, border, and margin. Understanding the box model from Week 4 was key to controlling spacing. Once I got this, a lot of layout problems I was having suddenly made sense.

Comments in CSS

Week 4 also taught us how to write comments in both HTML and CSS. In CSS, a comment looks like this: /* this is a comment */. I started using comments to label sections of my stylesheet so I could find things faster.

Tip: Use CSS variables for your colors and fonts at the top of your stylesheet. It makes changing your whole site's color scheme a one-line edit instead of hunting through every rule.

Flexbox and CSS Grid

Flexbox

Flexbox was covered in Week 4 and it completely changed how I think about layout. Before learning it, I was using margins and padding to try to line things up — which was a mess. With display: flex, you can align items in a row or column and control spacing with just a few properties. I used it for my navigation bar and for centering content on pages.

CSS Grid

CSS Grid is more powerful than Flexbox for two-dimensional layouts — meaning you control both rows and columns at the same time. I used display: grid with grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)) to build the card layout on my homepage. It automatically adjusts the number of columns based on screen width, which also helped with responsiveness.

Responsive Design

Week 12 focused on responsive design, which means making your site look good on phones, tablets, and desktop screens — not just one size. The main tool for this is the media query, which lets you apply different CSS rules depending on the screen width.

For example, on larger screens my site shows a multi-column grid. On screens under 768px wide, a media query switches it to a single column so nothing gets squished. I also used the viewport meta tag in the HTML <head> — without it, mobile browsers zoom out and ignore your responsive styles completely.
