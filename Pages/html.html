HTML Semantic Elements

Learning how to structure a webpage properly using HTML — the foundation of everything we built.

Starting with HTML

HTML was the first thing we learned in this course, starting in Week 2. Before this class, I had heard of HTML but had no idea how it actually worked. It stands for HyperText Markup Language, and it is basically the skeleton of every webpage — it defines what content exists and how it is organized, but not how it looks. That is CSS's job.

Every HTML file starts with the same basic structure: a <!DOCTYPE html> declaration, an <html> tag, a <head> section for metadata, and a <body> section for the actual content users see.

What Makes an Element "Semantic"?

This was one of the more important concepts from Week 2. A semantic element is one that describes the meaning of its content — not just how it looks on screen. For example, using <header> tells the browser and screen readers "this is the header of the page." Using a generic <div> does not communicate anything like that.

The difference matters a lot for accessibility and also for how search engines read your page. I started using semantic tags in all my assignments after learning this, and it genuinely makes your code easier to read too.

The Main Semantic Tags

  • <header> — The top section of the page, usually has the title or logo.
  • <nav> — Contains navigation links. Screen readers specifically look for this tag.
  • <main> — The primary content of the page. Should only appear once per page.
  • <section> — A standalone section of content, each with its own heading.
  • <article> — Self-contained content that could stand on its own, like a card or blog post.
  • <aside> — Supporting content that is related but not central, like a sidebar.
  • <footer> — The bottom of the page, usually has copyright info or extra links.

Good habit: Before you start building a page, sketch out which semantic tags you will use. It makes writing the actual HTML much faster and keeps your structure clean from the start.

Other HTML Basics From Week 2

Along with semantic structure, Week 2 covered the content elements I used in almost every assignment:

  • Headings — <h1> through <h6>. Only one <h1> per page, and they should go in order without skipping levels.
  • Paragraphs — <p> for any block of text.
  • Lists — <ul> for bullet points, <ol> for numbered lists, both using <li> for each item.
  • Links — <a href="url"> to link to other pages or external sites.
  • Images — <img src="path" alt="description">. The alt text is required for accessibility.
