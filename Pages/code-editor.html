Using the Code Editor

Getting started with Visual Studio Code — the tool I used for every assignment this semester.

Why Visual Studio Code?

When I first started this course, I had no idea what a code editor even was. I thought you could just write HTML in a regular text document. Turns out, there is a much better way — and that is Visual Studio Code, or VS Code for short.

VS Code is a free code editor made by Microsoft. It highlights your code in different colors, catches some errors as you type, and has a bunch of extensions that make writing HTML and CSS a lot easier. Once I set it up in Week 1, I used it for every single assignment after that.

How I Used It During the Semester

Setting Up My Project Folders

One of the first things I learned was how to organize my files properly. I kept my HTML files, CSS files, and images in separate folders so nothing got mixed up. VS Code makes this easy because you can open an entire folder and see all your files in one panel on the left side.

Features That Helped Me the Most

  • Emmet shortcuts — Typing ! and pressing Tab instantly creates a full HTML template. This saved me so much time.
  • Code highlighting — Different parts of the code show up in different colors, which makes it easier to read and spot mistakes.
  • Live Server extension — After installing this, I could right-click my HTML file and open it in a browser that automatically refreshes every time I save. No more manually reloading the page.
  • Prettier extension — This automatically formats and indents your code when you save. Kept everything clean without thinking about it.
  • IntelliSense — VS Code suggests tag names and property values as you type, which helped me when I could not remember the exact CSS property I needed.

Tip: Install the Live Server extension as soon as you set up VS Code. Being able to see your changes instantly in the browser makes everything so much faster.
