Validation for HTML and CSS

Using W3C validators and accessibility tools to catch errors and clean up code before publishing.

What Is Validation?

Validation means running your code through an official checker to see if it follows web standards correctly. Even if your page looks fine in Chrome, it might have errors that break it in other browsers or make it harder for assistive technologies to read.

We covered this in Week 13, which was one of the last weeks before the final project. At that point I had already written a lot of code, and running the validator showed me a few mistakes I had made without realizing — mostly small things like a missing closing tag or an attribute in the wrong place. Validation is one of those steps that feels optional until you actually do it and catch something.

Why Bother Validating?

  • Catches hidden errors — Things like unclosed tags or incorrect nesting that do not always show up visually but can cause problems.
  • Better browser compatibility — Valid code behaves more consistently across Chrome, Firefox, Safari, and Edge.
  • Helps with accessibility — Screen readers and assistive tools work better with clean, standards-compliant HTML.
  • Good professional habit — Running the validator before submitting or publishing is just part of doing the job right.

The Tools We Used

W3C HTML Validator

The W3C Markup Validation Service lets you check your HTML by pasting in your code, uploading a file, or entering a URL if your site is already published. It highlights the exact line number of each error and gives you a description of what is wrong.

W3C HTML Validator – validator.w3.org

W3C CSS Validator

Same idea but for your CSS stylesheet. You validate HTML and CSS separately. I found a couple of typos in my CSS this way that were stopping certain styles from applying.

W3C CSS Validator – jigsaw.w3.org

WAVE Accessibility Evaluation Tool

WAVE is a free tool that checks your live webpage for accessibility errors. It overlays icons directly on your page to show you exactly where the issues are — things like missing alt text, low contrast, or incorrect heading order. We used this in Week 13 alongside the HTML and CSS validators.

WAVE Web Accessibility Evaluation Tool – wave.webaim.org

Tip: Validate as you go, not just at the end. Fixing two errors on a page you just finished is much easier than finding twenty errors after you have built eight pages.

Common Errors to Watch Out For

  • Missing <!DOCTYPE html> at the very top of the file.
  • Unclosed tags — forgetting the closing </section> or </div>.
  • Images missing the alt attribute entirely.
  • Block elements nested inside inline elements incorrectly (like a <p> inside a <span>).
  • Using old, deprecated HTML tags like <center> or <font> instead of CSS.
  • Typos in CSS property names that cause styles to silently not apply.
